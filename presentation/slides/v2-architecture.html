1Ingestion — local, every 24h

Bluesky via atproto (search_posts) + GDELT news headlines only (titles), by keyword. A pure-Python topic filter (non-LLM) keeps only North-American extreme-climate content.

Bluesky posts older than 14 days are deleted (+ an availability sweep for author-deleted posts). GDELT is bounded separately (≤45d). SQLite holds every post + the classifications; ChromaDB holds only the GDELT evidence index.

2Classify — the only LLM decision

qwen2.5:3b, temperature 0, single-post (batch=1 → recall 0.938 vs ~0.75 batched — the biggest recall win was a config change, not a bigger model).

Prompt v3: persona + claim/opinion definitions in the system role; 8 few-shot examples + JSON format in the user message; a thought field placed first so it reasons before committing. Baked-in rule: "'no evidence provided' is never a reason to call something an opinion." Output {has_claim, reason}.

3Embed & index — build the RAG store

all-MiniLM-L6-v2 (384-dim). Chosen on a domain-pair eval over gte-small (flagged everything similar), mpnet (768-dim overkill), nomic (long-context we don't need) — separation 0.43 (similar 0.64 / dissimilar 0.21). Short text → semantic similarity, not long context.

Embeds the GDELT corpus into ChromaDB — the RAG index — with location folded into each document so same-region news ranks higher.

4Evaluate — two sets, on purpose

Gold = frozen 100 posts (48 claim / 52 opinion), never relabelled → catches regression. Dynamic = grows from admin relabels → catches concept drift.

Leak-free via in_eval_set / in_train_set columns (eval posts excluded from few-shot & the RAG index). Every run appends to eval_history.jsonl, stamped with model digest + runtime, so a move is a signal, not noise.

5Image eval — the multimodal path

A frozen image gold set (n=16) scores the image-gate + classify path — mean claim-text similarity 0.846.

Purpose: confirm the screenshot/vision entry doesn't regress the text pipeline — the image carries the claim, the text stays whatever it is.

6Maintain — self-healing + monitoring

apply_overrides replays the git-tracked admin_overrides.jsonl onto the labels; apply_provenance_labels stamps verbatim-headline posts to CLAIM.

Per-stage heartbeats to health.json → 🟢🟡🔴 chips in the app. DB flags written back: has_claim, admin_label, vision_signal. Historical evals appended to the drift log.

7FP image scan — precision recovery (parallel)

qwen2.5vl:7b on edge-case posts only: has_image & has_claim & bluesky, minus official / credible-cited. Decides image type, not truth.

A real photo depicting the claim → precision save (clears the red flag); a meme / cartoon / AI-suspected → reinforces it. A documented ~1% supplement, not the workhorse.

8Image-modality input — off-platform (parallel)

User uploads a screenshot of a Facebook / X / Instagram post. extract_from_image (qwen2.5vl:7b, 9-field schema) OCRs the claim, infers the platform from branding, reads visible engagement — transcribe / infer / null, never guess.

A thin adapter feeds the same assessment pipeline as source=uploaded_screenshot. Reaches where misinformation actually lives — but a screenshot strips the follower/verification data we score on and adds lossy OCR, so it's flagged a lower-confidence signal, by design.

9Retriever · RAG — the corroboration signal

At query time every assessed post — claim OR opinion — is embedded (with_location) and cosine-matched against the GDELT index; region-aware top-5. Opinions are retrieved too — that's what powers the "🔎 possibly a missed claim" nomination.

Verdict = proximity: a topical GDELT match reads "partial," never "corroborated" — topic overlap isn't event confirmation. But when a post links a credible source and that exact article is the top match, it surfaces as REPORTED — "the source this post cites reports this."

10Report formatter — reader signals

build_reader_signal assembles the reader view: evidence proximity + nearest sources, trust signals (official / self-cite / reshared-official / vision), the reach-vs-support red flag, and reframes (forecast / eyewitness).

No truth verdict, no credibility score — the reader draws the conclusion.

~2 min — the core pipeline segment. Four bands top to bottom: ① ingestion (local, daily); ② the Colab-GPU batch pass (classify → embed/index → eval → image-eval → maintain); ③ the vision layer — two image paths in parallel (FP scan for precision recovery, batch; screenshot input, live); ④ the live app (retrieve → report). Note the image layer is a parallel branch, not a step that feeds the retriever. Click each numbered box to open its card. Land two things: (1) the LLM's only job is classify — retrieval + rules do the rest; (2) opinions get embedded & retrieved too, which is how we catch a claim the classifier missed.
