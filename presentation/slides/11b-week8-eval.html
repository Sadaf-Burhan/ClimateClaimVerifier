[5:05–5:30] — ~25s. The evaluation layer the full-system slide pointed at.

The pipeline is only trustworthy if I can tell when it breaks. So Week 8 is the evaluation and monitoring layer.

There are two evaluation sets on purpose. A frozen gold set — 100 rows, never relabelled — catches regression. And a dynamic set that grows as the admin manually re-ranks and relabels borderline posts — that's what catches concept drift, the world changing under the model.

Everything is scored on one canonical backend — the Colab GPU; the app and command line can score too, but only as local diagnostics that don't write the log. Each run is appended to a drift log, stamped with the model version and runtime. That fingerprint is the whole point: it lets me tell a real metric move from run-to-run noise. It's how I know that gold sliding from 0.958 to 0.875 on a frozen set is a genuine regression, not variance — a finding I'll come back to in the results.

And health heartbeats from each stage drive green-amber-red chips in the app sidebar, so degradation shows up before a user notices.
