[2:30–3:05] — ~35s. A big week; two improvements then the reckoning.

Week 4 was a refactor plus two improvements. First, compute: with no local GPU, classification crawled on CPU, so I moved it to a free Colab T4 GPU — export the unclassified posts, classify on the GPU, import the results back. Ingestion stayed local. Remember this — it pays off in a second.

Second, prompt v3: task and claim/opinion definitions in the system role, eight few-shot examples plus a chain-of-thought thought field in the output. Recall went from 0.792 to 0.833. Two small findings: the format example has to stay in the user message — moving it to system dropped recall to 0.71 — and eight examples beat ten, which made the model too conservative.

Then evaluation got honest and found two bugs in the measurement, not the model. Leakage: six of my eight few-shot were themselves eval posts — I was showing the model the test answers, which inflated the numbers. And batch drift: classifying sixteen posts per call drifted the 3B model into under-calling claims.

[click] The fix was leak-free examples plus single-post inference — recall to 0.938. And here's the payoff from the GPU: single-post is about sixteen times more calls, which is only affordable on the GPU. So the compute decision and the accuracy decision are the same decision. The biggest recall win in the whole project was a config change, not a bigger model — which left precision, about 0.69, as the real open problem for Week 5.
